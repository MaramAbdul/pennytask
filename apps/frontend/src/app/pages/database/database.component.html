ğŸ‘¤ Users Collection (Schema & Model)

This is the **User schema** used in our project.

import { Prop, Schema, SchemaFactory } from '@nestjs/mongoose';
import { Document } from 'mongoose';

export type UserDocument = User & Document;

@Schema()
export class User {
  @Prop({ required: true })
  name: string;

  @Prop({ required: true, unique: true })
  email: string;

  @Prop({ required: true })
  password: string;
}

export const UserSchema = SchemaFactory.createForClass(User);

ğŸ” Sessions Collection (User Login Sessions)

This schema is used to manage user sessions and store JWT tokens.

@Schema()
export class Session {
  @Prop({ required: true })
  userId: string;

  @Prop({ required: true })
  token: string;

  @Prop({ default: () => new Date(Date.now() + 8 * 60 * 60 * 1000) })
  expiresAt: Date;
}

export const SessionSchema = SchemaFactory.createForClass(Session);

ğŸ”¹ Store a New Session Token

When a user logs in, we store their session token.

  userId: user._id, 
  token: 'jwt-token-here'

Used in **AuthGuard** to validate user sessions.
